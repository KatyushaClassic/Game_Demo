# 飞船爬塔 Roguelike 设计草案

## 1. 核心循环

1. 进入某层大地图，选择下一个节点（战斗/事件/商店/维修/精英/Boss）。
2. 进入战斗后操控飞船：机动、武器组开火、技能释放。
3. 战斗结束获得方块、资源、评分增量。
4. 战斗外进行改装与合成，调整武器组与特殊方块。
5. 击败层 Boss 进入下一层，共 4 层。
6. 结算评分，进入局外成长，解锁新方块和核心。

## 2. 大地图（4 层）

- 每层固定一个终点 Boss 节点。
- 中间由多条路径连接，房间类型可控权重生成。
- 建议房间类型：
  - 普通战斗
  - 精英战斗
  - 事件
  - 商店
  - 维修/补给
  - 宝箱
  - Boss

### 2.1 生成约束（建议）

- 第一层前两行不出现精英。
- 同一路径上连续商店不超过 1 次。
- Boss 前一行至少有一次维修或事件点。

## 3. 飞船拼装系统

## 3.1 方块稀有度

- 普通、稀有、史诗、传说。
- 传说来源：Boss 掉落、合成产出。
- 核心不参与稀有度分层。

## 3.2 方块类型

1. 掩体（防御、承伤、结构连接）
2. 炮塔（火炮/导弹/点防，含无限与有限弹药）
3. 推进器（方向性推力，正推 100%，反推 40%）
4. 核心（被毁即失败，开局选择）
5. 特殊（替代遗物，需安装在船体上）

## 3.3 尺寸与占位

- 方块支持多尺寸（1x1、2x2、3x3...）。
- 放置规则：
  - 不可越界、不可重叠。
  - 必须与已存在结构连通（核心作为起点）。

## 4. 操作与战斗

- `WASD`：平移/推进。
- 鼠标方向：武器目标朝向。
- 对有射界限制的武器，超界时自动夹角到最近可射方向。
- 主动技能：键位 `1~4` + UI 按钮点击。
- 支持技能自动释放策略（例如 CD 完成且有敌时自动施放）。

## 4.1 武器组

- 玩家可建立多个武器组并切换。
- 可设置某些组由玩家主控，某些组自动开火。
- 离开改装界面前，若武器变化，必须确认武器组配置。

## 5. 特殊方块系统

- 分为被动与主动。
- 被动可能是全局增益或战斗内触发。
- 主动挂接到技能栏。
- 支持安装条件（例如“2 格范围内至少 2 门火炮”）。
- 支持冲突规则（装配 A 后禁装 B）。

## 6. 合成系统

- 战斗外任意时刻可打开合成。
- 按配方消耗方块生成新方块。
- 合成不可撤销。
- 部分强力方块仅可通过合成获得。

## 7. 局外成长

- 每局结束按表现结算评分。
- 评分作为成长货币，用于升级。
- 升级解锁：新方块池、新核心、可能的新房间事件。

## 8. 技术落地建议（Godot）

- 数据驱动优先：
  - 方块、配方、核心、事件建议用 `Resource` 或 JSON 配置。
- 系统分层：
  - 数据层：`BlockData`、`RecipeData`
  - 逻辑层：拼装/武器组/合成/地图/成长
  - 表现层：战斗场景、拼装 UI、地图 UI
- 可测试性：
  - 拼装合法性、合成校验、武器组路由等逻辑优先写无场景依赖单元测试。
